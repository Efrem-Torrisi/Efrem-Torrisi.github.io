Auto LOD Tool

Batch LOD and Convex Collision Mesh Generator for Houdini

Role: Solo Developer Tools: Houdini, Python, Unreal Engine 5 Type: 3D Pipeline Tool

Overview & Technical Breakdown

For this project, I built a Houdini shelf tool that automates the generation of Level of Detail meshes and convex collision geometry from a source FBX mesh. The tool takes an input mesh, produces LOD0 through LOD3 by reducing polygon count at user-defined percentages, and optionally generates UCX collision meshes in either a simple or complex mode. The PolyReduce step preserves UVs where possible, and in cases where reduction distorts them, a UV Transfer pass can be used to restore the original mapping. Everything is exported as a single FBX file with correct naming conventions so that engines like Unreal Engine 5 automatically recognize the LOD chain and collision meshes on import.

Network Overview

When exectued, the tool creates all the needed 'obj' and 'out' node chains needed in an organized subnetwork. The 'obj' networks's top-level network is organized into 1/2 additional subnetworks and 1/2 temp geo nodes depending on user settings. The source_file node holds the imported FBX input. The LODS subnetwork contains the LOD generation chain, producing LOD0 through LOD3 at the user's specified reduction percentages. The UCX subnetwork handles collision mesh generation in either simple or complex mode. A temporary complex collision node sits alongside for the complex mode before the results are split into individually named collision pieces.

LOD Generation

Inside the LODS subnetwork, four geometry networks generate the LOD chain. Each LOD node merges in the source mesh and applies a PolyReduce with a user-defined settings and target percentage. The defaults are 100% for LOD0 (the original mesh), 50% for LOD1, 25% for LOD2, and 15% for LOD3, and settings for preserving quads and/or recomputing normals, these can be adjusted by the user each time they want to run the tool. Each geo network outputs the reduced mesh with the correct naming convention (e.g. meshname_LOD0, meshname_LOD1) so that the final FBX export will be ready for engine import.

Inside an LOD Node

Each individual LOD node follows the same simple pipeline: an Object Merge brings in the source mesh, a PolyReduce node with the user defined settings that reduces it to the target percentage, and a Null marks the output. This structure is replicated across all four LOD levels, with only the reduction percentage and the preserve quad setting if enabled changing between them.

Collision Generation: Simple Mode

In simple mode, the tool generates a single convex collision mesh for the entire input. The LOD0 mesh is brought in via Object Merge and passed through a Shrinkwrap node to produce a convex hull that tightly wraps around the original shape. This is fast and well-suited for meshes that are already roughly convex or where a single collision volume is sufficient. The result is named with the UCX_ prefix so that Unreal Engine automatically recognizes it as a collision mesh.

Collision Generation: Complex Mode

For meshes with concave or intricate shapes, the complex mode uses Convex Decomposition to break the mesh into multiple convex pieces. The user can set a maximum convexity level to control the accuracy-to-count tradeoff: higher values produce more collision pieces that better approximate the original shape, while lower values keep the count down for performance. Each resulting piece is isolated into its own geometry node inside the UCX subnetwork and named sequentially (e.g. UCX_meshname_00, UCX_meshname_01) following the UE5 naming convention.

UCX Subnetwork

After the Convex Decomposition splits the mesh, each resulting convex piece gets its own geometry node inside the UCX subnetwork. The number of nodes scales with the user's chosen convexity level. Each piece follows the same internal pipeline: merge in from the decomposed result, isolate its specific segment, and output with its sequential UCX_ name. All collision pieces are then included in the final FBX export alongside the LOD chain.

Result & Export

The final output is a single FBX file containing the full LOD chain (LOD0 through LOD3), the source mesh, and all generated UCX collision meshes. The naming convention is applied automatically so that Unreal Engine 5 recognizes the LOD levels and collision volumes on import without any manual setup.
