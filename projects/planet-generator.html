Overview & Technical Breakdown

A Blender add-on tool that procedurally builds terrestrial planets from a sidebar panel. Each planet is generated as its own collection of spheres (planet, clouds, atmosphere) along with a Sun light and a tracking camera for rendering. Multiple planets can exist in the same scene and be edited independently through the side panel.

All three materials are made with the shader nodes that have important values wired to the addon UI and update in real-time. The tool also exports meshes as FBX and can bakes albedo, roughness, normal, and a packed cloud albedo+alpha via the Cycles renderer for game-ready use.

Material Graphs

Planet Surface

Two FBM noises feeding each other, then ramps turn the result into color, roughness, and normals.

Clouds

Two noises: one drives the alpha for cloud shape, the other drives the color, so shape and color can be tuned independently.

Atmosphere

A thin volume shader for the outer shell with controls for denisty via the color ramp, and color.

Game-Ready Export & Bake

Mesh export writes the active planet's icospheres to an FBX in a single click. Texture bakes run on Cycles at a the selected resolution and format (PNG or TGA), producing albedo, roughness, and normal maps for the surface, with an optional spherical UV reproject for meshes that don't already have these UVs.

Cloud baking is a two-step process: a standard DIFFUSE bake captures the cloud color, then the shader graph is temporarily rewired so the cloud alpha ramp drives an Emission shader that gets baked as the opacity pass. The two results are merged in-memory into a single RGBA PNG (color in RGB, opacity packed into A) so the cloud sphere can be reused in real-time engines as one transparent texture.
