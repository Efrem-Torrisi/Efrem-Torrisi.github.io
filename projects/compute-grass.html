Overview & Technical Breakdown

An Unreal Engine 5 plugin that renders millions of grass blades entirely on the GPU. A compute shader handles spawning, culling, and sorting blades into LOD buckets, while custom global vertex and pixel shaders generate the curved geometry procedurally and write directly to the GBuffer so it integrates with Unreal’s deferred rendering pass.

Planned for easy drop-in usage: add a Grass Volume actor to the scene and configure wind, colors, height/width variations, bending, and dithering.

Compute Shader and Indirect Drawing

The core of the plugin is a compute shader that runs one GPU thread per potential grass blade. Each thread evaluates a spawn position on a grid, performs frustum and distance culling, and if the blade is visible, appends it to a structured buffer.

The draw arguments are written entirely on the GPU into an indirect args buffer, so the CPU never needs to know how many blades are visible. The final draw call uses DrawPrimitiveIndirect to render only the instances that passed all checks, achieving zero CPU-GPU synchronization overhead.

[numthreads(64, 1, 1)]
void MainCS(uint3 DispatchThreadId : SV_DispatchThreadID)
{
    uint ThreadIndex = DispatchThreadId.x;

    // Only thread 0 needs to write the draw args
    if (ThreadIndex == 0)
    {
        OutDrawArgsLOD0[0].VertexCountPerInstance = VertexCountPerInstanceLOD0;
        OutDrawArgsLOD0[0].StartVertexLocation   = 0;
        OutDrawArgsLOD0[0].StartInstanceLocation  = 0;

        OutDrawArgsLOD1[0].VertexCountPerInstance = VertexCountPerInstanceLOD1;
        OutDrawArgsLOD1[0].StartVertexLocation   = 0;
        OutDrawArgsLOD1[0].StartInstanceLocation  = 0;
    }

    if (ThreadIndex >= NumGrassBlades)
        return;

    // Get a grid position and jitter it so the blades don't look too uniform
    uint GridSize = uint(sqrt(float(NumGrassBlades)));
    float2 GridUV = IndexToGridPosition(ThreadIndex, GridSize);

    float JitterX = Hash(ThreadIndex * 2) * 0.8 - 0.4;
    float JitterY = Hash(ThreadIndex * 2 + 1) * 0.8 - 0.4;
    GridUV += float2(JitterX, JitterY) / float(GridSize);
    GridUV = saturate(GridUV);

    // Convert grid UV to world space
    float3 WorldPos;
    WorldPos.x = SpawnOrigin.x + (GridUV.x * 2.0 - 1.0) * SpawnExtent.x;
    WorldPos.y = SpawnOrigin.y + (GridUV.y * 2.0 - 1.0) * SpawnExtent.y;
    WorldPos.z = SpawnOrigin.z;

    // If circular mode is on, skip blades outside the circle
    if (bCircularShape)
    {
        float2 offset = (WorldPos.xy - SpawnOrigin.xy) / SpawnExtent.xy;
        if (dot(offset, offset) > 1.0)
            return;
    }

    // Skip if too far or not in view
    float DistanceToCamera = length(WorldPos - CameraPosition);
    if (DistanceToCamera > MaxDrawDistance)
        return;
    if (!IsInFrustum(WorldPos))
        return;

    // If we got here the blade is visible, so fill in its data
    FGrassInstance Instance;
    Instance.Position = WorldPos;
    Instance.Scale    = 0.8 + Hash(ThreadIndex * 3) * 0.4;
    Instance.Normal   = float3(0, 0, 1);
    Instance.Rotation = Hash(ThreadIndex * 4) * 6.28318;

    // Put it in the close or far LOD buffer and increment the count
    uint WriteIndex;
    if (DistanceToCamera <= LODDistance)
    {
        InterlockedAdd(OutDrawArgsLOD0[0].InstanceCount, 1, WriteIndex);
        OutGrassInstancesLOD0[WriteIndex] = Instance;
    }
    else
    {
        InterlockedAdd(OutDrawArgsLOD1[0].InstanceCount, 1, WriteIndex);
        OutGrassInstancesLOD1[WriteIndex] = Instance;
    }
}
// One buffer per LOD to store the grass instances that pass culling
FRDGBufferDesc InstanceDesc = FRDGBufferDesc::CreateStructuredDesc(
    sizeof(FGrassInstanceGPU), MaxInstances);

FRDGBufferRef InstancesLOD0 = GraphBuilder.CreateBuffer(InstanceDesc, TEXT("GrassInstancesLOD0"));
FRDGBufferRef InstancesLOD1 = GraphBuilder.CreateBuffer(InstanceDesc, TEXT("GrassInstancesLOD1"));

// These hold the draw args - the compute shader fills in how many to draw
FRDGBufferDesc ArgsDesc = FRDGBufferDesc::CreateStructuredDesc(
    sizeof(FGrassDrawIndirectArgsGPU), 1);
ArgsDesc.Usage |= EBufferUsageFlags::DrawIndirect;

FRDGBufferRef ArgsLOD0 = GraphBuilder.CreateBuffer(ArgsDesc, TEXT("GrassArgsLOD0"));
FRDGBufferRef ArgsLOD1 = GraphBuilder.CreateBuffer(ArgsDesc, TEXT("GrassArgsLOD1"));

// Clear to zero before the shader runs
AddClearUAVPass(GraphBuilder, GraphBuilder.CreateUAV(ArgsLOD0), 0);
AddClearUAVPass(GraphBuilder, GraphBuilder.CreateUAV(ArgsLOD1), 0);

// Pass in spawn area, camera info, and culling settings
auto* PassParams = GraphBuilder.AllocParameters<FGrassComputeCS::FParameters>();
PassParams->NumGrassBlades = MaxInstances;
PassParams->SpawnOrigin    = FVector3f(Settings.SpawnOrigin);
PassParams->SpawnExtent    = FVector3f(Settings.SpawnRadius, Settings.SpawnRadius, 0.f);
PassParams->CameraPosition = CameraPositionF;
PassParams->MaxDrawDistance = Settings.MaxDrawDistance;
PassParams->LODDistance     = Settings.LODDistance;
ExtractFrustumPlanes(InView, PassParams);

// Give the shader write access to the instance and draw arg buffers
PassParams->OutGrassInstancesLOD0 = GraphBuilder.CreateUAV(InstancesLOD0);
PassParams->OutGrassInstancesLOD1 = GraphBuilder.CreateUAV(InstancesLOD1);
PassParams->OutDrawArgsLOD0       = GraphBuilder.CreateUAV(ArgsLOD0);
PassParams->OutDrawArgsLOD1       = GraphBuilder.CreateUAV(ArgsLOD1);

// Kick off the compute shader - one thread per blade
const uint32 GroupSize   = 64;
const uint32 GroupCount  = FMath::DivideAndRoundUp(MaxInstances, GroupSize);

FComputeShaderUtils::AddPass(
    GraphBuilder,
    RDG_EVENT_NAME("GrassCompute_Culling"),
    ComputeShader, PassParams,
    FIntVector(GroupCount, 1, 1));

// ... VS/PS setup, pipeline state init ...

// Draw with indirect args - the GPU already knows how many blades there are
RHICmdList.DrawPrimitiveIndirect(DrawParams->IndirectDrawArgs->GetRHI(), 0);

Vertex and Pixel Global Shaders

Custom global shaders write directly to the GBuffer. The vertex shader generates curved blade geometry procedurally from SV_VertexID with bending driven by wind, per-blade random tilt, droop, and side curvature.

The pixel shader outputs a base-to-tip color gradient with blade hue variation, root self-shadowing, world-space noise patches, Interleaved Gradient Noise dithered opacity for soft tips, and a subsurface scattering color for backlit translucency.

Optimization Techniques

Frustum and distance culling run as thread checks in the compute pass, so the indirect draw call automatically renders fewer instances. A two-level LOD system sorts blades into separate buffers: nearby blades get full 15-vertex curved geometry, distant blades fall back to a 3-vertex triangle. Screen-space width clamping prevents sub-pixel artifacting.

The plugin hooks into PostRenderBasePassDeferred, so all grass participates in deferred lighting, shadows, and GI without a separate lighting pass.

Plugin and User Parameters

A self-contained UE5 plugin. Users drop a GrassVolume actor and configure blade count, spawn radius, draw distance, height/width, bending, wind, colors, LOD distance, and dithering via exposed UProperties. Multiple independent volumes are supported, each dispatching its own compute and draw passes.
