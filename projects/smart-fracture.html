Smart Mesh Fracturing Tool

Automated per-material mesh pre-fracturing for Houdini

Role: Solo Developer Tools: Houdini, Python, Unreal Engine 5 Type: 3D Pipeline Tool

Overview & Technical Breakdown

For this project, I developed a Houdini HDA tool that automates the pre-fracturing pipeline for FBX meshes. It auto-detects materials on an imported mesh via naming convention through a Python SOP, applies per-material fracturing methods, then reassembles the fractured parts with correct naming conventions for a hassle-free export back to a game engine or alternatively for use with RBD/Bullet solvers.

Material Auto-Detection

When using the tool in its "Auto Detect Materials Mode", the tool's Python SOP reads the incoming FBX geometry and parses material and name attributes to identify each material zone, and then creates primitive and detail attributes used for export naming convention later. Each detected material is separated into its own processing stream, where it receives the appropriate fracture treatment based on its material type and user parameters.

Per-Material Custom Fracture Methods

A big part of this project was exploring how to fracture meshes to get realistic results for different materials (concrete, wood, glass). Throughout development of this tool I developed numerous custom fracturing methods for these three materials and eventually ended up including in the tool three variations for concrete, three for glass, and two for wood. The tool applies the correct fracturing method per material zone, producing physically plausible pre-fracture patterns without manual setup requested from the user. Alternatively, if the user wants to, the tool can be used in a manual mode by forcing a specific fracturing method to be applied to the entirety of the mesh.

Reassembly & Naming Conventions

After fracturing, the tool reassembles all fragments into a single hierarchy with standardized naming. There are two different naming patterns, one that preserves the parental hierarchy of meshes, if any existed in the input mesh, and one that gives a simple piece index for each disconnected mesh. Material attributes are also preserved to facilitate automation of material application in engines such as Unreal Engine.

Export Modes & Settings

The tool offers multiple export modes: a single combined FBX that keeps the offset for pieces relative to the original pivot, and an FBX export with all pieces centered at the world origin. Both modes allow the user to transfer input mesh UVs to the pre-fractured mesh, triangulate, remesh, or polyreduce before export.

Point Cloud & Metadata Export

The tool can generate two additional exports alongside the fractured meshes. The first is a point cloud FBX that stores each piece's original offset from the mesh pivot, allowing an engine to reconstruct the intact shape from origin-centered pieces. The second is a JSON metadata file that maps each piece name to its position and bounding box dimensions, providing everything needed to generate correctly placed collision boxes in code.

Destruction Demo in Unreal Engine 5

To demonstrate a practical use case for the tool, I built a lightweight destruction system in Unreal Engine. The pre-fractured pieces are exported centered at the world origin, alongside a point cloud FBX that stores each piece's original offset from the mesh pivot. I then made a GPU Niagara system using Mesh Renderer Flipbook to render each fractured piece as a particle, with an Initial Mesh Location module reading the point cloud FBX to reconstruct the original shape of the intact mesh and some point and rotation forces applied for the effect.

A Blueprint then holds the original intact mesh and the Niagara system. On projectile hit, the intact mesh is destroyed and the Niagara system is activated -- the particles start at their reconstructed positions and simulate outward, producing a satisfying destruction effect. Because the particle count equals the number of fractured pieces, the system is extremely performant while still delivering a convincing result.

Destruction Demo in Custom Engine (Jolt Physics)

The fractured meshes were also used in a custom DX11 engine with Jolt physics for real-time destruction. For this setup, the origin-centered FBX pieces and the JSON metadata export were used -- the engine reads each piece's position and dimensions from the JSON to generate collision boxes at the correct placement and scale in code. On impact, the intact mesh is swapped for the individual pieces with their physics bodies, and Jolt simulates the destruction from there.
