Sketch2Level

2D Pixel Sketch to 3D Level Assembly Pipeline

Role: Solo Developer Tools: Houdini, Python, Unreal Engine 5 Type: Level Design Pipeline Tool

Overview & Technical Breakdown

For this project, I built a Houdini HDA that converts top-down 2D sketches into assembled 3D levels. The input is one or more PNG images where each pixel represents a tile, and different pixel colors map to different modular kit pieces: floors, walls, doors, windows, stairs, pillars, corners, ecc... The tool reads the image, automatically detects where walls and corners need to be placed based on floor adjacency, and outputs a point cloud with type, rotation, and floor index attributes. These points are then used to instance the corresponding modular mesh pieces, producing a complete level layout preview in houdini from a simple pixel sketch. The tool supports multi-floor levels by stacking multiple sketch images, and exports a JSON manifest with coordinate conversion for Unreal Engine, Unity, or any custom axis system. Any program that can draw at a single-pixel level and export PNGs works as the sketching tool, though the dedicated standalone app shown in the demo is a work in progress.

Color Legend and Pixel Parsing

Each pixel color in the input PNG maps to a building element type. The legend is fully user-configurable through the HDA's parameters, with default colors for floor, wall, corner, door, window, stair, pillar, and empty space. A Python SOP reads the image using PIL (Python Imaging Library), iterates over every pixel, and checks it against the legend to give the point the appropriate type attribute value. Floor pixels and special types like stairs and pillars are placed directly as points at their grid positions, while wall positions are inferred based on floors positions in a later pass. The wall color used in the sketch is used to indicate which type of wall should be attached to the corresponding floor tile.

Wall Placement and Corner Detection

For each floor pixel, the tool checks its four cardinal neighbors. If a neighbor is a wall pixel (wall, door, or window color), a wall piece is placed at the edge between the two cells with the correct rotation. The wall pixel's color determines whether the placed piece is a standard wall, a door, a window, empty or any other modular piece used in your kit with minimal modifications of the HDA. The tool then performs a pass to detect corner conditions: where two perpendicular walls meet, the tool classifies the junction as an inner L-corner, outer L-corner, T-junction, or X-junction based on how many walls converge and how many floor tiles surround that point. Inner corners consume their adjacent wall edges to avoid overlap.

Multi-Floor Support

The tool supports multi-floor levels through the HDA multiparm input floors PNGs parameter. Each floor takes its own PNG image, and the tool processes them sequentially, offsetting each floor's points by a procedural amount vertically depending on the height of your modular kit's wall piece. This means a level with multiple vertical levels can be sketched as a series of simple top-down drawings, one per floor, and the tool stacks them into a stacking 3D structure.

JSON Export

Once the level is assembled, the HDA can export a JSON manifest containing every placed piece's mesh name, position, rotation, and scale. The export supports coordinate conversion for compatibility with engines that use different coordinate systems.

Unreal Engine Import

A Python script runs inside Unreal Engine's editor via an Editor Utility Widget to import the level data JSON. The script reads the json, checks each piece's mesh name against a configurable asset path, and, if meshes with corresponding names exist, spawns the Static Mesh actors at the correct positions and rotations. Because the JSON export already handles the coordinate conversion from Houdini's coordinate system to Unreal's system, the import script can place everything directly without additional transform logic. The result is a folder with the level fully assembled, ready for lighting, set dressing, and gameplay inteagration. Additionally, since each piece is spawned as its own independent actor, the result is identical to a level built by hand, level designers can freely move, delete, or replace any individual piece after import, making it easy to use the assembled layout as a starting point and manually iterate from there.
