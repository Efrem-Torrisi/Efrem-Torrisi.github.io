Compute Shader Grass Plugin

GPU-Driven Grass Plugin via Compute Shaders in Unreal Engine 5

Role: Solo Developer Tools: Unreal Engine 5, C++, HLSL Year: 2025

Overview & Technical Breakdown

For this project, I developed an Unreal Engine 5 plugin that renders millions of individual grass blades entirely on the GPU via compute shaders and indirect drawing. A compute shader spawns, culls, and sorts blades into LOD buckets, while custom global vertex and pixel shaders generate curved blade geometry procedurally and write directly to the GBuffer, integrating with Unreal's default deferred rendering pass. The plugin is designed to be a drop-in solution: place a Grass Volume actor in the scene and configure parameters like wind animation, base/tip/subsurface colors, height and width variations, bending, rotation, and dithering to fit the desired art style.

Compute Shader and Indirect Drawing

The core of the plugin is a compute shader that runs one GPU thread per potential grass blade. Each thread evaluates a spawn position on a grid with hash-based jitter, performs frustum and distance culling, and if the blade is visible, appends it to a structured buffer. The draw arguments are written entirely on the GPU into an indirect args buffer, so the CPU never needs to know how many blades are visible. The final draw call uses DrawPrimitiveIndirect to render only the surviving instances, achieving zero CPU-GPU synchronization overhead.

Vertex and Pixel Global Shaders

The grass blades are rendered using custom global shaders that write directly to the GBuffer. The vertex shader generates blade geometry procedurally from SV_VertexID, building curved blades with Bezier bending driven by wind, per-blade random tilt, droop, and side curvature. The pixel shader outputs a base-to-tip color gradient with per-blade hue variation, self-shadowing at the roots, world-space noise based patches that vary in color value, Interleaved Gradient Noise dithered opacity for soft blade tips, and a subsurface scattering color for backlit translucency.

Optimization Techniques

Several layers of optimization keep the plugin within a reasonable frame budget at high blade counts. On the compute side, frustum culling discards blades outside the camera view and distance culling rejects blades beyond a configurable draw distance. Both are simple per-thread checks before the append, so that the indirect draw call automatically renders fewer instances. A two-level LOD system sorts blades into separate buffers: nearby blades get the full 15-vertex curved geometry while distant blades fall back to a 3-vertex triangle, each drawn with their own indirect call. Screen-space width clamping enforces a minimum pixel width on distant blades to prevent artifacting. On the rendering side, the plugin hooks into PostRenderBasePassDeferred, meaning all grass participates in deferred lighting, shadows, and GI for "free" without requiring a separate lighting pass.

Plugin and User Parameters

The plugin is structured as a self-contained UE5 runtime module. When the plugin is enabled, the user can drop a GrassVolume actor with uproperty parameters available for editing, such as: blade count, spawn radius, draw distance, height, width, bending and rotation variations, wind speed and direction, base/tip/subsurface colors, LOD distance, dithering settings and frustum culling margin. The plugin supports multiple independent grass volumes in a scene, each with its own settings, dispatching separate compute and draw passes per volume.
