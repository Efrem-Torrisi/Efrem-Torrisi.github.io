Overview & Technical Breakdown

A Houdini shelf tool that automates LOD mesh generation and convex collision geometry from a source FBX. The tool produces LOD0 through LOD3 at user-defined reduction percentages and optionally generates UCX collision meshes in simple or complex mode.

PolyReduce preserves UVs where possible, with a UV Transfer fallback for cases where reduction distorts them. Everything is exported as a single FBX with naming conventions that engines like Unreal Engine automatically recognize on import.

Workflow Demo

Network Overview

When executed, the tool creates organized 'obj' and 'out' node chains. The top-level network contains the source FBX, a LODS subnetwork for LOD generation, and a UCX subnetwork for collision mesh generation.

LOD Generation

Four geometry networks generate the LOD chain, each applying PolyReduce at a user-defined target percentage (defaults: 100%, 50%, 25%, 15%). Settings for preserving quads and recomputing normals are configurable per run.

Each output is named with the correct convention (e.g. meshname_LOD0, meshname_LOD1) for automatic engine recognition on import.

Inside an LOD Node

Each LOD node follows the same pipeline: Object Merge brings in the source mesh, PolyReduce reduces to the target percentage, and a Null marks the output. This structure is replicated across all four levels with only the reduction settings changing.

Collision Generation: Simple Mode

Simple mode generates a single convex hull via Shrinkwrap around the LOD0 mesh. Fast and well-suited for roughly convex shapes or where one collision volume is sufficient. Named with the UCX_ prefix for automatic engine recognition.

Collision Generation: Complex Mode

For concave or intricate shapes, complex mode uses Convex Decomposition to split the mesh into multiple convex pieces. A maximum convexity parameter controls the accuracy-to-count tradeoff. Each piece is named sequentially (e.g. UCX_meshname_00, UCX_meshname_01) following the UE5 convention.

UCX Subnetwork

Each convex piece gets its own geometry node inside the UCX subnetwork, following the same pipeline: merge in the decomposed result, isolate the segment, and output with its UCX_ name. All pieces are included in the final FBX alongside the LOD chain.

Result & Export

The final output is a single FBX containing the full LOD chain, source mesh, and all UCX collision meshes with automatic naming conventions for engine import.
