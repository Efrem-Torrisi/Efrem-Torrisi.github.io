Smart Mesh Fracturing Tool

Automated per-material mesh pre-fracturing for Houdini

Overview & Technical Breakdown

For this project, I developed a Houdini HDA tool that automates the pre-fracturing pipeline for FBX meshes. It auto-detects materials on an imported mesh via naming convention through a Python SOP, applies per-material fracturing methods, then organizes the fractured parts with correct naming conventions for a hassle-free export back to a game engine or alternatively for use with RBD/Bullet solvers.

Workflow Demo

Material Auto-Detection

In "Auto Detect Materials Mode", a Python SOP parses the incoming FBX's material and name attributes to identify each material zone. Each zone is then separated into its own processing stream and fractured according to its material type and user parameters.

Per-Material Custom Fracture Methods

The tool includes custom fracturing methods for concrete (3 variations), glass (3), and wood (2), each producing physically plausible break patterns. In auto mode the correct method is applied per material zone; in manual mode the user can force a specific method across the entire mesh.

Reassembly & Naming Conventions

After fracturing, fragments are merged back together with standardized naming: either preserving the original parental hierarchy or using a flat piece index. Material attributes are preserved for automated material assignment in-engine.

Export Modes & Settings

Two export modes are available: a combined FBX with pieces offset relative to the original pivot, or an FBX with all pieces centered at the world origin. Both support UV transfer, triangulation, remeshing, and polyreduction before export.

Point Cloud & Metadata Export

Two additional exports can be generated: a point cloud FBX storing each piece's original offset from the mesh pivot (for reconstructing the intact shape), and a JSON file mapping each piece name to its position and bounding-box dimensions (for reconstruction or collision box generation in code).

Destruction Demo in Unreal Engine 5

To demonstrate a practical use case, I built a lightweight destruction system in UE5. A GPU Niagara system uses Mesh Renderer Flipbook to render each fractured piece as a particle, reading the point cloud FBX via an Initial Mesh Location module to reconstruct the intact shape.

On projectile hit, the intact mesh is destroyed and the Niagara system activates, simulating pieces outward. Since the particle count equals the piece count, performance stays very high.

Destruction Demo in Custom Engine (Jolt Physics)

The tool's exports were also used in a custom DX11 engine with Jolt physics. The engine reads piece positions and dimensions from the JSON to generate collision boxes, and on impact swaps the intact mesh for the physics-enabled pieces.
