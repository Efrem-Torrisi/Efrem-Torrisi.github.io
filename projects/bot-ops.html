My Contributions

Player Rig

I built the first-person player rig in Blender, driving both weapons and the camera. The rig includes two full weapon hierarchies, dedicated bones for projectile and muzzle flash spawn points, and bones for camera control, all organized with Copy Transforms/Location/Rotation constraints and custom bone shapes.

The camera bone matches the engine's field of view exactly, so what I see in Blender is a 1:1 match with the in-game perspective.

Player Animations

I animated all first-person actions for both weapons: idle, walk, fire, reload, alternative reload, and weapon swap transitions.

Destructible Objects

I designed the destructible objects pipeline with the programming team. Props are pre-fractured using my Smart Mesh Fracturing Tool, which exports an FBX alongside a JSON file mapping each piece's position and bounding-box dimensions.

At load time the engine reads the JSON to generate collision boxes, and on impact swaps the intact mesh for the individual pieces with Jolt physics bodies to simulate the destruction.

Level Data Validator

Some objects in the level data files would occasionally be saved with absurdly large translation values, silently crashing the level on startup. Finding them manually was impractical.

I wrote a Python script that scans every entry, flags transforms outside a sane range, and lets you delete or inspect them directly.

Pickup VFX

I created pickup VFX inspired by Call of Duty Zombies power-ups, written entirely in HLSL. The vertex shader drives the bobbing and rotation, while the pixel shader handles the glowing, pulsating outline and color shifts.
