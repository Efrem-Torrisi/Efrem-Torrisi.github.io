My Contributions

Player Rig

I built the first-person player rig in Blender, which drives both weapons and the camera. The rig includes two full weapon hierarchies, dedicated bones for projectile and muzzle flash spawn points used by the gameplay programmers, and bones for controlling the in-game camera. To keep everything controllable, the rig makes heavy use of modifiers such as Copy Transforms, Copy Location, and Copy Rotation constraints, along with custom bone shapes for intuitive selection in the viewport. The camera bone was set up to match the exact field of view used in the engine so that what I see in Blender is a 1:1 match with the in-game perspective.

Player Animations

I animated all first-person actions for both weapons: idle, walk, fire, reload, alternative reload, and weapon swap transitions.

Destructible Objects

I collaborated with the programming team to design the destructible objects pipeline and logic, and supported its implementation. Specific props in the environment can be destroyed by the player, breaking apart into pre-fractured pieces with Jolt physics. The meshes were pre-fractured using my Smart Mesh Fracturing Tool, which exports the fractured pieces in an FBX alongside a JSON metadata file mapping each piece's name to its position and bounding-box dimensions. The engine reads this JSON at load time to generate correctly placed and scaled collision boxes in code, and on impact swaps the intact mesh for the individual pieces with their physics bodies so Jolt can simulate the destruction from there.

Level Data Validator

During production, we ran into an issue where some objects in the level data files would be saved with extremely large translation values (coordinates with 30+ zeroes), silently crashing the level on startup with no clear error. Tracking these entries down manually was impractical given the size of the data files. I wrote a Python script that parses every entry in the level data files, flags any transforms with values outside a sane range, and presents them in a list. From there you can either delete the bad entries directly or open them for inspection, turning what was a painful debugging session into a couple of clicks fix.

Pickup VFX

I created the in-game pickup visual effects, inspired by the power-up style from Call of Duty Zombies. The effect was written entirely in HLSL, working on both the vertex and pixel shaders. The vertex shader handles the bobbing and rotation animation of the pickup mesh, while the pixel shader produces the glowing, pulsating outline and color shifts that make pickups visually distinct and readable from a distance.
